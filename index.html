Clips Share

This is a video sharing application. It has a database implemented with Firebase that stores the users data and the videos uploaded by the users. After creating a new account, users are able to upload their own videos and to manage them. During the process of uploading a video, the user can choose a thumbnail from a selection of screenshoots generated using a WebAssembly / JavaScript port of FFmpeg. The website is hosted on Vercel and was made using Angular and RxJS.

Clothing Store

This app is a clothing store. It has a database implemented with Firebase and a payment method implemented with Stripe. The database stores the users data and the app data. After creating a new account, users can access their profile, where they can modify their address and see their orders history. Items can be added to the cart. The shopping cart data is saved inside the local storage so that it can be accessed later. At checkout users can go to the payment page where they can make payments using a test credit card. After the user clicks 'payment', the order data is saved into the user's orders history inside Firebase. Payments can also be made without being authenticated, in which case the order data is not saved. The website is posted on Netlify and uses a severless function for requesting payment intents from Stripe. The app is made with React and the state is managed with Redux and Redux Saga.

Smart Brain

This is a website created in React that can detect faces. It uses an API from a deep learning AI platform named Clarifai. After making an account, you can copy-paste an image link containing a jpg photo and the app with the help from the API will detect faces in your picture. The front-end part of the app is hosted on Render and the back-end is hosted on Fly.io. The back-end part has access to a database stored in a PostgreSQL instance on Fly. The database has two tables created, one for storing users and one for storing login info where the passwords are stored encrypted. It takes a few seconds for the website to access the back-end after the first request, because Fly disables the back-end when not in use!

Space Launches API

This is a back-end RESTful API that serves a front-end application that can launch space missions. The API is made in Express and it's connected to a MongoDB database. Every time someone launches a mission in the front-end, the mission data is stored in the database. For the destination planet input in the front-end, the API parses a CSV file with planets data discovered by NASA and filters the planets that may be habitable. It also uses a paginated SpaceX API to get all the historical/ upcoming rocket launches scheduled by SpaceX. These launches are also saved in the database so that they can be listed in the front-end. The Server API is deployed as a Docker Web Service hosted on Render.com.
